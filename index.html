初始 Git

Git 是一个开源的分布式版本控制系统。

【Git 官网】、【Pro Git】、【廖雪峰Git教程】

一、Git 安装

1、Windows上安装

用于开发环境。

【官方Git】、【TortoiseGit】

推荐使用TortoiseGit，含GUI界面以及SSH密钥工具（开始->TortoiseGit->puttygen）。

2、Linux上安装

用于生产环境。

参见Linux环境配置篇。

二、Git 环境初始化

windows下通过TortoiseGit设置，主要是通过puttygen生成设置ssh.ppk。

下面主要讲解linux下的git环境设置。

git config命令专门用来设置环境，环境参数可以放在以下三个地方：

  • /etc/gitconfig作用于所有用户，对应git config --system
  • ~/.gitconfig作用于当前用户，对应git config --global
  • 项目目录中的.git/config仅针对当前项目，对应git config

设置用户信息：

查看配置信息：git config --list

三、SSH密钥

用于本地与平台之间建立安全的加密连接

  • 默认存放在~/.ssh目录（看隐藏文件ls -a）
  • 私钥id_rsa：自己保留，与平台加密通讯的钥匙
  • 公钥id_rsa.pub：提供给平台，用于授权验证

1、生成密钥

linux下由SSH包提供工具；
windows下由 git bash中MSysGit包提供；

2、提供公钥

找到公钥文件，将内容复制并添加到平台的ssh列表。

3、测试连接

ssh -T git@github.com

4、TortoiseGit配置

TortoiseGit需要通过PuTTY进行ssh通讯。

  • 安装PuTTY软件
  • 通过PuTTYGen生成.ppk格式的密钥、公钥文件
  • 在TortoiseGit中设置引入.ppk密钥文件
  • 将.ppk公钥文件中的内容，添加到平台的ssh列表

四、入门术语

1、通过基本流程理解工作区域：

  1. 远程仓库`remote repository
    1. 通过git clone下载到本地库
  2. 本地仓库`local repository
    1. 工作目录working directory
      1. 未被管理文件、目录
        1. 通过git add加入暂存区
      2. 暂存区域staging area
        1. 通过git commit提交到本地仓库
      3. 本地仓库local repository
        1. 通过git push推送到远程仓库
      4. 远程仓库remote repository

2、理解 git clone 和 fork 的区别

git clone 从远程库拷贝到本地，但是对于别人的项目，你无法更改推送回远程库。

fork 从远程库拷贝到自己的远程库，并作为原库的一个分支，可以进行修改，并可 pull request 提出合并到原库的请求。

3、仓库被克隆到本地后，git都做了什么？

  • 将远程库在本地命名为origin
  • 并建立指向远程库master分支的指针，在本地命名为origin/master
    • 因为远程库是在不断更新的，这个指针代表本地更新到那步了
    • 下次与远程库通讯时，会下载新内容，此指针也会更新
  • 建立属于你自己的本地master分支
    • 所以就有了本地master分支推送到origin/master远程库分支的说法

1、分支的应用

  • master：只用于发布生产环境
  • dev：用于主线开发
    • 测试通过后合并到master分支
    • 注意：过程中要随时拉取及合并master分支，以便及时得到别人的修改
  • iss：主线开发中处理突发异常
    • 主线开发工作没有结束时，从master分支衍生出处理突发异常的分支
    • 完成后合并到master分支（dev会从master中获取到本次修改）
